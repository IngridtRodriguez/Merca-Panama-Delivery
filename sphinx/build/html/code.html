Documentación del código¶

El proyecto tiene una estructura de templates y static. Dentro de templates se encuentra todos los html y en statics se encuentra CSS, JS e imagenes. A continuación, se muestra el código .py principal en la raíz del proyecto.

Modulo principal¶

UserLogin_Merca.about()[fuente]¶

Ruta de contacto. Se entra en ella poniendo en url localhost/contacto

Devuelve:template contacto.html
UserLogin_Merca.carrito()[fuente]¶

Ruta para el carrito. Se entra a ella con localhost/cart Esta función se encarga de mostrar todos los artículos y cantidades guardadas en cierta sesión. Desde esta página se lleva a la pagina de pago al presionar un botón de tramitar.

Devuelve:template cart.html
UserLogin_Merca.checkout()[fuente]¶

Esta función se encarga del pago de la orden del carrito. Tiene dos formularios: uno de actualización de datos y dirección de vivienda y otro de Pago Al actualizar datos de la dirección de vivienda se abre el Collection usuarios y edita estos campos. Al pagar la base de datos resta la cantidad de artículos disponibles en el Collection articulos y borra el carrito del usuario.

Devuelve:template checkout.html
UserLogin_Merca.index()[fuente]¶

Ruta de la pagina principal, se ingresa a ella escribiendo la ruta localhost/

Devuelve:template index.html con todos los artículos en la base de datos
UserLogin_Merca.login()[fuente]¶

Ruta para el inicio de sesión. Se ingresa a ella con localhost/login Se especifica los métodos utilizados ya que aquí se obtendrá datos si se presiona un submit La función login utiliza método POST para detectar los request a ingreso de datos.

Devuelve:template login.html
Devuelve:error
UserLogin_Merca.para_todos()[fuente]¶

Esta función se encarga de inyectar variables automaticamente en el contexto de los templates (html’s). Pasa el subtotal (con un formato) de la persona logueada a todas las paginas Esto es fundamental para el caso del header.html, ya que el header tiene el icono de carrito Header.html se repite en todas las páginas, asi solo se programa un vez y solo es incluido en otro html (por ejemplo, index)

UserLogin_Merca.registrar()[fuente]¶

Ruta para el registro, se entra a ella con localhost/register Maneja entrada de datos, se especifica método POST

Devuelve:template register.html
Devuelve:error
UserLogin_Merca.shop_grid(categoria_en_seleccion)[fuente]¶

Ruta para catalogo, se ingresa a ella con localhost/shop-grid/. Por default abre las frutas Utiliza URL converter, lo que da la capacidad de crear un URL dinámico. Es decir, se utiliza parte del URL como una variable de búsqueda de productos por categoría Por default la categoría en selección son las Frutas.

Parámetros:categoria_en_seleccion – Variable Dinamica
Devuelve:template shop-grid.html
Devuelve:categoria_en_seleccion
UserLogin_Merca.single_product(articulo)[fuente]¶

Ruta de los productos. Se ingresa a ella con localhost/producto/Piña o localhost/producto/Naranja o desde la página de catalogo Justo como con el catalogo, utiliza URL converter para pasar URL como variables (En este caso, el artículo) <articulo> se pasa desde el shop-grid donde se le dé click a un producto

También utiliza método POST en caso de Añadir un producto al carrito

Parámetros:articulo – Variable dinamica
Devuelve:template single-product.html
Devuelve:articulo
